# hack5 · 生态与可持续规划

## 使命

> **让开源社区生态可持续地发展。**
>
> 方式:不断为这个生态**提供更多的数字公共物品(Digital Public Goods)**来帮助它。
> 这里的"生态"指的是**开源社区生态**;hack5(黑客松平台)只是我们提供的**第一个**公共物品,后面还会有更多。

这份文档不是为了商业化,而是让 hack5 在开源社区生态中发挥更多价值:

- 我们是**公共物品**——大多数人应该能**简单、免费**地办一次黑客松;
- 只有当持续使用、真实消耗我们的 token 与资源时,才**适度、定量**地收费;
- 长期主义:目标不是赚很多钱,而是让这个开源社区生态**可持续地运转下去**,并靠它自身的积累(开发者、社区、媒体网络)反哺更多公共物品。

## 一、定位与哲学

| 原则 | 含义 |
|---|---|
| **公共物品** | 办一次黑客松,核心功能全开、永久免费。记录不删。 |
| **成本触发收费** | 只有真实产生成本的地方才收费:AI 生成、邮件量、多场持续运营、自定义域名。 |
| **长期主义** | 不追求利润最大化,追求生态可持续 + 社区网络的复利。 |
| **限量共享** | 我们积累的资源与社会网络对大家开放,但**限量**——用得起、也养得起。 |

## 四、我们真正的资产(护城河)

不只是代码和编程能力。围绕社区慢慢形成的东西,才是复利:

1. **开发者 email 名单** —— 每个人只要参加过一次黑客松,就留下了 email。这是**持续积累的基础**,随每场活动增长。
2. **多社区共同主办的网络** —— 我们会和很多社区一起办黑客松,沉淀出跨社区的协作关系。
3. **可共享(限量)的资源与社会网络** —— 场地、导师、赞助渠道、媒体触达,可以为参与者所共享。
4. **自有媒体矩阵** —— 公众号、小红书、微信群、Telegram 群;这是触达开发者、也是给赞助商看价值的凭证。

> 这些是收费之外、让生态可持续的真正底座:钱只是覆盖成本,**网络与名单才是长期价值**。

## 五、可持续模型(商业方向)

分三层,对齐"免费为主、成本触发收费":

### 1. 免费层(获客底座)
单场黑客松,核心功能全开(建站/报名/评审/作品墙/组队/免费海报/一键转发)。永久免费。

### 2. 消耗触发层(Pro)—— 有成本才收费
触发点都是**真实成本**:
- 第 2 场起 / 持续运营
- AI 类:AI 海报(gpt-image-1)、未来 AI 评审/摘要 —— token 直接花钱
- 邮件量(Resend 付费资源)
- 自定义域名 / 去标白标

**定价锚点(初步,先不锁死)**:**100 元 / 10 场活动**。

### 3. 赞助层(生态 B 端)
- **赞助方案**:**1500 美元 / 年,限 4 席**(或 1000 元 / 10 年的社区档)
- **权益**:所有 hack5 组织的黑客松页面 + 子域名页脚带 **sponsor 标**
- **传播兑现**:多平台订阅用户一起转发,触达 **~3000** 开发相关领域的关注者/开发者
- **媒体页**:展示自有媒体矩阵的触达价值(见下)

## 六、媒体页(赞助触达的凭证)

一个专门页面,展示"我们拥有的媒体和粉丝数",作为赞助定价与触达的依据。每个渠道列出:

| 字段 | 说明 |
|---|---|
| 账号 | 具体账号名 |
| 平台 | 公众号 / 小红书 / 微信群 / Telegram 群 |
| 粉丝数 | 当前规模 |
| 主要 topic | 这个账号主要讨论什么 |
| 链接 | 对应的可点击链接 |

> 需要你提供各渠道的真实数据(账号/粉丝数/topic/链接),我才能把这一页做出来。

## 七、路线建议(优先级)

**已完成并上线**:多租户建站、报名+日程、评审、作品墙、Turnstile、建站邮件、免费海报、AI 海报(付费)、一键转发、组队墙、品牌资产(OG/favicon)、主办方组织资料。

**下一步建议(A → B → C)**:

- **A. 赞助地基(先能谈钱)**:全站/页脚 sponsor 标 + 媒体页。有了展示位和触达凭证,才能开始谈赞助。
- **B. 传播放大(提升触达、反哺赞助价值)**:Telebot 转发 / 可订阅黑客松 Bot。
- **C. 参赛者体验闭环**:证书自动生成、赛后反馈问卷、奖项登记。
